## 题目原文

### 题目一：语音版的日历工具

请开发一个语音日历工具，帮助用户提高日历管理的效率和便捷性。

**要求**：请了解用户在日历管理中的真实需求，设计并实现一个以语音交互为核心的日历管理工具，能准确、顺畅地实现通过语音添加/删除/查看事件提醒等能力。

## 作品有效性

1. 作品需符合所选议题方向，确保自主完成无抄袭
2. **全周期持续交付**：从议题发布之日起保持持续的 PR 和 commit 记录。仅在最后一天一次性导入所有代码 → 直接无效
3. 所有 commit 时间戳必须落在批次时间（5/29 00:00 – 5/31 23:59）之内
4. 其他无效情形：
   - PR 描述空白或与实际代码变更严重不符
   - 引用第三方库未在 README 列明依赖且未说明原创部分
   - 复用自己过去的代码片段未在 PR 描述中注明来源

## PR 提交规范

1. 基于 PR 添加新功能
2. **每个 PR 只做一件事**：鼓励粒度尽可能小的 PR，大功能拆分为多个独立 PR
3. PR 标题与描述需包含：
   - **标题**：一句话说明新增/修改了什么
   - **功能描述**：作用与使用方式
   - **实现思路**：技术选型或核心逻辑
   - **测试方式**：如何验证正常运行
4. PR 合并后主分支必须可运行

## Demo 视频要求

1. 用声音完整讲解、展示作品主要功能和效果，覆盖核心模块
2. 上传至 bilibili/云盘等外部平台，链接放 README **显眼位置**
3. 不要藏在文件夹/子文件夹内

## 其他要求

- 若多人组队，每队仅提交一个代码仓库。队员均须各自提交 commit，PR 清晰描述分工
- 多模块提交至同一仓库不同子目录（如 `/frontend` 与 `//backend`）
- 代码仓库需在开题后创建
