# VocaCal UI 设计文档

> 定位：温暖友好、语音优先的日历管理工具
> 用途：定义 UI 视觉规范、页面结构、交互流程，供竞品参考和 Google Stitch 原型制作

---

## 1. 设计语言

### 1.1 视觉风格关键词

温暖、友好、清晰、信任感、不压迫

- 不是冷冰冰的工具，而是像一个贴心的语音助手
- 配色温暖但不过于鲜艳，避免「玩具感」
- 信息密度适中——日历 App 需要一览性，但不能拥挤

### 1.2 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| **主色** | `#6C5CE7`（柔和紫） | 按钮、选中态、强调元素 |
| **主色浅** | `#A29BFE` | 渐变、悬浮按钮光晕、录音脉冲 |
| **辅色** | `#FDCB6E`（暖黄） | 今天标记、重要事件高亮 |
| **成功** | `#00B894`（清新绿） | 添加成功反馈 |
| **警告** | `#E17055`（暖橙） | 删除确认、错误提示 |
| **背景** | `#FAFAFA` | 主背景 |
| **卡片** | `#FFFFFF` | 卡片、底部弹窗 |
| **文字主** | `#2D3436` | 标题、正文 |
| **文字辅** | `#636E72` | 次要信息、时间 |

### 1.3 圆角与阴影

| 元素 | 圆角 | 阴影 |
|------|------|------|
| 卡片 | 16px | `0 2px 12px rgba(0,0,0,0.08)` |
| 按钮 | 12px | 无（用色块） |
| 底部弹窗 | 顶部 24px | `0 -4px 24px rgba(0,0,0,0.12)` |
| 语音按钮 | 50%（圆形） | `0 4px 20px rgba(108,92,231,0.4)` |
| 头像/图标 | 8px | 无 |

### 1.4 字体

| 用途 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 24sp | Bold |
| 卡片标题 | 18sp | Medium |
| 正文/事件标题 | 16sp | Regular |
| 时间/次要信息 | 14sp | Regular |
| 标签/提示 | 12sp | Regular |

### 1.5 动画原则

- **录音脉冲**：麦克风按钮外圈从中心向外扩散的波纹，紫色渐变，周期 1.2s
- **状态切换**：300ms ease-out，不要突兀弹跳
- **卡片出现**：从底部滑入 + 淡入，200ms
- **成功反馈**：短暂的绿色对勾动画 + TTS 语音
- **避免**：过多装饰性动画、旋转、闪烁

## 2. 页面结构

### 2.1 主屏幕（唯一页面）

```
┌─────────────────────────────────────┐
│  ☀️ VocaCal                    ⚙️   │  ← 标题栏：App名 + 设置入口
├─────────────────────────────────────┤
│                                     │
│       ◀  2026年5月  ▶              │  ← 月份切换器
│                                     │
│  一   二   三   四   五   六   日    │
│                                     │
│  25  26  27  28  29⦿ 30  31        │  ← 日历网格，今天有圆点标记
│                                     │  ← 选中日期有填充色
│  1   2   3   4   5   6   7         │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  5月29日 周四  ·  今天              │  ← 选中日期标题
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🕐 15:00    开会             │   │  ← 事件卡片
│  │             项目周会          │   │
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🕐 19:00    健身             │   │
│  └─────────────────────────────┘   │
│                                     │
│         （暂无日程，试试语音添加）     │  ← 空状态
│                                     │
├─────────────────────────────────────┤
│                                     │
│              🎙️                     │  ← 悬浮语音按钮（FAB）
│         按住说话                     │    居中偏下，醒目但不遮挡
│                                     │
└─────────────────────────────────────┘
```

**关键设计决策**：

1. **日历网格紧凑**：只占屏幕上方 ~40%，留足够空间给事件列表
2. **语音按钮 FAB 式**：底部悬浮，不占据事件列表空间，点击后展开为底部弹窗
3. **事件卡片左对齐时间 + 右对齐标题**：扫一眼就能看到时间线
4. **空状态有引导文案**：「暂无日程，试试语音添加」—— 降低首次使用门槛

## 3. 核心交互流程 UI

### 3.1 语音录音态（底部弹窗展开）

用户按住麦克风按钮 → 底部弹窗从下往上展开，覆盖事件列表下半部分：

```
┌─────────────────────────────────────┐
│  （日历网格仍然可见）                 │
│  5月29日 周四                        │
├─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
│                                     │
│         ╭───────────╮              │
│         │  ◉◉◉◉◉◉  │              │  ← 脉冲动画：紫色波纹
│         │    🎙️     │              │
│         │  正在聆听  │              │
│         ╰───────────╯              │
│                                     │
│    "明天下午三点开会"                  │  ← ASR 实时识别文字（大字号）
│                                     │
│    ┌─ 提示 ─────────────────────┐  │
│    │ 试试说：                    │  │
│    │ · 明天下午三点开会           │  │
│    │ · 这周末有什么安排           │  │
│    │ · 把明天的开会删掉           │  │
│    └────────────────────────────┘  │
│                                     │
│          [ 取消 ]                    │
│                                     │
└─────────────────────────────────────┘
```

**要点**：
- 弹窗高度约 60% 屏幕，日历网格保持在顶部可见
- 识别文字用 20sp 大字号显示，让用户确认 ASR 结果
- 底部提示文案帮助新用户了解支持的说法
- 录音中麦克风按钮变为红色脉冲

### 3.2 添加成功反馈

ASR 识别 + NLU 解析后，弹窗内容变为结果展示：

```
┌─────────────────────────────────────┐
│                                     │
│    ✅ 已添加                         │  ← 成功图标 + 文字
│                                     │
│    ┌─────────────────────────────┐  │
│    │ 📅 5月30日 周六              │  │  ← 解析后的事件卡片
│    │ 🕐 下午 3:00                │  │     带绝对日期 + 星期
│    │ 📝 开会                      │  │
│    └─────────────────────────────┘  │
│                                     │
│    🔊 "已添加5月30日周六下午3点开会"   │  ← TTS 播报文字同步显示
│                                     │
│          [ 确认 ]                    │  ← 点击关闭弹窗
│                                     │
└─────────────────────────────────────┘
```

**要点**：
- 回复文字里带绝对日期 + 星期，用户能一眼判断对不对
- 视觉（绿色对勾 + 卡片）+ 听觉（TTS）双重反馈
- 2 秒后自动关闭，或用户点击确认关闭

## 4. 组件清单

### 4.1 VoiceButton（语音按钮）

| 状态 | 外观 | 动画 |
|------|------|------|
| **待命** | 紫色圆形 FAB，白色麦克风图标 | 微微呼吸（scale 1.0 → 1.05，周期 2s） |
| **录音中** | 红色圆形，外圈紫色脉冲波纹扩散 | 脉冲波纹，1.2s 周期 |
| **处理中** | 紫色圆形，内部 loading spinner | 旋转动画 |

尺寸：64×64dp，距底部 32dp 居中

### 4.2 EventCard（事件卡片）

```
┌──────────────────────────────────┐
│  🔵 15:00              开会       │  ← 左侧彩色竖条标识
│                    项目周会        │     时间左对齐，标题右对齐
└──────────────────────────────────┘
```

- 白色背景，16px 圆角，轻阴影
- 左侧 4px 竖条（颜色可按时间段区分：上午蓝、下午紫、晚上靛）
- 点击可展开查看详情

### 4.3 CalendarGrid（月历网格）

- 星期标题行：一~日，灰色 12sp
- 日期格子：40×40dp，选中态紫色填充+白字，今天黄色圆点
- 有事件的日期底部加 2dp 的小圆点（紫色）
- 滑动左右切换月份

### 4.4 BottomSheet（底部弹窗）

- 白色背景，顶部 24px 圆角
- 下拉手柄（小横条）可手动收起
- 三种内容状态：录音中 / 结果展示 / 错误提示

## 6. 设计约束

| 约束 | 原因 |
|------|------|
| 单屏幕 MVP，不做多页面导航 | 减少开发量，2.5 天够用 |
| 日历组件用现成库（`react-native-calendars`） | 不造轮子 |
| 动画优先级：录音脉冲 > 卡片滑入 > 其余 | 录音反馈是核心体验 |
| 配色不超过 5 种主色 | 避免杂乱 |
| 不做深色模式 | MVP 不需要 |
| 不做横屏适配 | 日历 App 竖屏为主 |
