# VocaCal 竞赛开发约束

本项目参加「七牛云 × XEngineer 暑期实训营」第二批次议题一（语音日历工具），所有开发决策必须符合以下规则。

## 评审标准（指导优先级）

| 维度 | 占比 | 行动指引 |
|------|------|----------|
| **作品完整度与创新性** | 40% | 功能必须完整（增/删/查全覆盖），交互流畅，鼓励创新点 |
| **开发过程与质量** | 40% | 架构清晰、代码规范可读、PR 数量与质量、commit 分布合理 |
| **演示与表达** | 20% | 最终需录 Demo 视频（声音讲解 + 功能展示），链接放 README 显眼位置 |

### 开发优先级推导

1. **核心功能优先**：语音添加/删除/查看事件（占 40% 评审的最大块）
2. **代码质量必须高**：好的架构 + 规范代码 = 另一个 40%
3. **交互体验要流畅**：语音反馈、UI 美观、操作顺畅
4. **创新加分项**：在核心功能完成后，可考虑智能提醒、多语言、日程建议等

## 无效作品红线（绝对不能碰）

- ❌ **最后一天突击提交**：仅在最后一天一次性导入所有代码 → 直接无效
  - ✅ 必须：从开发第一天起保持持续的 commit 和 PR 记录
- ❌ **commit 时间戳超出批次时间**：所有 commit 必须在 5/29 00:00 – 5/31 23:59 之间
- ❌ **PR 描述空白或与实际变更不符**
- ❌ **代码重复率超过 50%**
- ❌ **代码或技术抄袭**
- ❌ **使用了第三方库但没在 README 列明**

## 架构要求

### 目录结构

```
VocaCal/
├── app/        ← React Native 前端
├── server/     ← FastAPI 后端
├── docs/       ← 项目文档
├── README.md
└── .env.example
```

- 多模块在同一仓库不同子目录（frontend 与 backend 分别管理）
- 便于部署和启动

### 代码规范

- **可读性**：变量/函数命名清晰，关键逻辑加注释
- **健壮性**：错误处理完善，语音识别失败/网络异常有优雅降级
- **规范性**：统一的代码风格，lint 通过
- **主分支随时可运行**：评委可能在任意时间查看，main 分支代码必须能跑起来

## README 必须包含

- [ ] 项目简介和功能说明
- [ ] Demo 视频链接（显眼位置，不能藏在子文件夹）
- [ ] 技术栈说明
- [ ] 所有第三方依赖及版本列表
- [ ] 快速启动命令（前后端分别说明）
- [ ] 环境变量说明（.env.example）

## 时间约束

- **实战期**：5月29日 00:00 – 5月31日 23:59
- **仓库公开**：6月1日 00:00 起必须设为公开
- **每天必须提交**：至少 1 个有效 PR，保持 commit 分布合理
